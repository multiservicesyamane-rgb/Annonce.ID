## 3. ARCHITECTURE SEO DES URLS

### 3.1 Structure d'URL — Règles absolues

- ✅ Toujours en minuscules
- ✅ Séparateur : tiret `-` (jamais `_`)
- ✅ Sans paramètres dynamiques (pas de `?id=123`)
- ✅ Maximum 3 niveaux de profondeur
- ✅ Mots-clés dans l'URL
- ❌ Jamais de stop-words inutiles (le, la, de, un…)

### 3.2 Schéma d'URLs recommandé

```
# Accueil
https://annonceswest.com/

# Catégories principales (pages evergreen haute priorité)
```

## 5. SEO TECHNIQUE

### 5.1 Sitemap XML

Générer automatiquement via Next.js / sitemap package :

```xml
<!-- sitemap.xml — structure recommandée -->
<?xml version="1.0" encoding="UTF-8"?>
<sitemapindex>
  <sitemap>
    <loc>https://annonceswest.com/sitemap-pages.xml</loc>     <!-- pages statiques -->
  </sitemap>
  <sitemap>
    <loc>https://annonceswest.com/sitemap-categories.xml</loc> <!-- toutes catégories -->
  </sitemap>
  <sitemap>
    <loc>https://annonceswest.com/sitemap-villes.xml</loc>    <!-- pages ville×catégorie -->
  </sitemap>
  <sitemap>
    <loc>https://annonceswest.com/sitemap-annonces.xml</loc>  <!-- annonces actives (paginated) -->
  </sitemap>
  <sitemap>
    <loc>https://annonceswest.com/sitemap-blog.xml</loc>      <!-- articles -->
  </sitemap>
</sitemapindex>
```

**Règles du sitemap annonces :**
- N'inclure que les annonces avec `status = 'active'`
- Mettre à jour la `<lastmod>` à chaque modification
- Exclure les annonces expirées immédiatement
- Utiliser `<changefreq>daily</changefreq>` pour les listings
- Limiter à 50 000 URLs par fichier sitemap (pagination si nécessaire)

### 5.2 Robots.txt

```txt
User-agent: *
Allow: /
Disallow: /yamanetech/
Disallow: /dashboard/
Disallow: /paiement/
Disallow: /connexion
Disallow: /inscription
Disallow: /api/
Disallow: /*?*sort=*
Disallow: /*?*page=*&sort=*
Disallow: /recherche?*

# Autoriser les bots de médias sociaux
User-agent: facebookexternalhit
Allow: /annonce/

User-agent: Twitterbot
Allow: /annonce/

# Sitemap
Sitemap: https://annonceswest.com/sitemap.xml
```

### 5.3 Balises canoniques — Gestion des doublons

```html
<!-- Sur toutes les pages filtrées (tri, pagination) -->
<!-- Page 2 de listing -->
<link rel="canonical" href="https://annonceswest.com/immobilier/dakar/">

<!-- Paramètres à exclure de l'index -->
?sort=prix_asc → canonical vers URL sans paramètre
?page=2 → utiliser rel="next" / rel="prev"
?vue=liste → canonical vers URL de base
```

### 5.4 Balises Hreflang (multilangue)

À implémenter dès la version EN/Wolof :

```html
<link rel="alternate" hreflang="fr" href="https://annonceswest.com/immobilier/dakar/">
<link rel="alternate" hreflang="en" href="https://annonceswest.com/en/real-estate/dakar/">
<link rel="alternate" hreflang="x-default" href="https://annonceswest.com/immobilier/dakar/">
```

### 5.5 Redirections & Gestion des erreurs

```
# 301 — Redirections permanentes (URL restructurée)
/category/immo → /immobilier/ (301)
/ads/12345     → /annonce/12345/slug (301)

# Pages d'erreur optimisées
404 → Page personnalisée avec barre de recherche + suggestions populaires
410 → Pour les annonces expirées (signale à Google que c'est définitif)
```

### 5.6 En-têtes HTTP SEO

```
# Via Next.js next.config.js ou Cloudflare
X-Robots-Tag: noindex  → sur /dashboard/, /yamanetech/
Cache-Control: public, max-age=31536000  → assets statiques
Cache-Control: public, s-maxage=3600, stale-while-revalidate → pages ISR
```

## 10. PERFORMANCE & CORE WEB VITALS

### 10.1 Objectifs Core Web Vitals (Google Search Console)

| Métrique | Seuil « Bon » | Objectif AnnoncesWest |
|---------|--------------|----------------------|
| **LCP** (Largest Contentful Paint) | < 2.5s | < 2.0s |
| **FID / INP** (Interaction to Next Paint) | < 200ms | < 150ms |
| **CLS** (Cumulative Layout Shift) | < 0.1 | < 0.05 |
| **FCP** (First Contentful Paint) | < 1.8s | < 1.5s |
| **TTFB** | < 800ms | < 400ms |

### 10.2 Optimisations images (critique pour le SEO)

```html
<!-- Image principale annonce : priorité high -->
<img
  src="/images/annonce-12345-cover.webp"
  alt="Villa F5 avec piscine à Almadies, Dakar — 280 000 000 FCFA"
  width="800"
  height="600"
  loading="eager"
  fetchpriority="high"
  srcset="
    /images/annonce-12345-cover-400.webp 400w,
    /images/annonce-12345-cover-800.webp 800w,
    /images/annonce-12345-cover-1200.webp 1200w
  "
  sizes="(max-width: 768px) 100vw, 60vw"
>

<!-- Images non-critiques (hors viewport) -->
<img loading="lazy" decoding="async" ...>
```

**Formats à utiliser :**
- WebP pour tous les navigateurs modernes
- AVIF pour les navigateurs compatibles (30% plus léger que WebP)
- JPEG fallback pour les vieux navigateurs
- Compression : qualité 80-85 pour JPEG/WebP

### 10.3 Optimisations Next.js spécifiques

```javascript
// next.config.js
module.exports = {
  images: {
    formats: ['image/avif', 'image/webp'],
    deviceSizes: [640, 750, 828, 1080, 1200],
    minimumCacheTTL: 2592000, // 30 jours
  },
  // ISR pour les pages catégorie
  // revalidate: 3600 (1h) pour les pages listing
  // revalidate: 300 (5min) pour les nouvelles annonces
}
```

### 10.4 Stratégie de cache

```
# Pages statiques (catégories, pages pays)
Cache-Control: public, s-maxage=3600, stale-while-revalidate=86400

# Pages annonces (contenu qui peut changer)
Cache-Control: public, s-maxage=300, stale-while-revalidate=600

# API (données dynamiques)
Cache-Control: public, s-maxage=60, stale-while-revalidate=120

# Assets statiques (JS, CSS, images)
Cache-Control: public, max-age=31536000, immutable
```

## 11. SEO MOBILE-FIRST

### 11.1 Vérification Mobile-Friendliness

AnnoncesWest étant mobile-first (85% de l'audience), voici les points critiques :

**Technique :**
```html
<!-- Viewport obligatoire -->
<meta name="viewport" content="width=device-width, initial-scale=1">

<!-- Pas de contenu masqué sur mobile qui est visible sur desktop -->
<!-- Google indexe la version mobile EN PREMIER -->
```

**UX mobile pour le SEO :**
- Texte lisible sans zoom (minimum 16px)
- Boutons suffisamment grands (minimum 48×48px)
- Pas de contenu plus large que l'écran
- Pas d'interstitiels intrusifs sur mobile (pénalité Google)

### 11.2 PWA & SEO

## 12. SUIVI & KPIs

**Actions à faire au lancement :**
1. Vérifier le domaine via DNS TXT record
2. Soumettre le sitemap : `https://annonceswest.com/sitemap.xml`
3. Surveiller les erreurs d'indexation hebdomadairement
